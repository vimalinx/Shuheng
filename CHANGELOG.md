# Changelog

## 0.2.0a1 - Experimental Local Alpha - 2026-07-10

- Added editable, immutable-build Pi-native Agent Projects behind the permanent
  OMP main Agent, including explicit Tool grants, task-policy separation,
  Run Manifest provenance, single-writer participation, and non-replayable
  recovery contracts.
- Moved model credentials into Shuheng-owned private state and tightened file,
  backup, subprocess-environment, and OMP approval boundaries.
- Added truthful OMP/Pi runtime setup and doctor behavior for clean machines.
- Reduced the persistent JSONL gateway to a positive four-action public schema
  and removed internal metadata and machine paths from its frames.
- Made the release reproducible on declared Python/build-tool versions and
  added clean-clone CI coverage without maintainer-only sibling repositories.
- Changed release identity from the original `0.1.0` package to PEP 440
  `0.2.0a1`, with fresh-install and alpha.1-upgrade verification.
- Removed maintainer-local Trellis workflow state from the public Git source
  boundary and moved Shuheng-owned contributor contracts into ordinary docs.
- Refreshed security, install, upgrade, uninstall, contribution, release, and
  third-party dependency documentation.

## 0.1.0 - Experimental Local Alpha - 2026-07-04

- Initial Shuheng package metadata and command entry points.
- Local curses TUI for sessions, task ledgers, artifacts, approvals, Secret
  Vault, runtime providers, gateway metadata, scheduler surfaces, and
  compatibility bridges.
- Linux-first fresh-machine install guidance with WSL2 recommendation for
  Windows users, macOS best-effort wording, and native Windows unsupported
  status.
- Bundled shared `shuheng-agent-gateway` skill installer for local stdio agent
  discovery and governed message dispatch instructions.
- Release hardening for private local knowledge bases, public install docs,
  release hygiene, and wheel/sdist smoke contracts.
- Public alpha release candidate posture with release hygiene, wheel/sdist
  smoke, fresh-install expectations, GitHub templates, and explicit known gaps.
- Release posture is experimental local alpha. The supported gateway is local
  JSONL stdio; A2A/MCP-shaped records, heuristic eval, baseline comparison, and
  scheduler automation remain experimental local surfaces.
